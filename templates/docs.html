StockClip AI Pro REST API

Welcome to the official documentation for the StockClip AI Video Engine (V16.2). This backend microservice provides an asynchronous, highly optimized pipeline to fetch, analyze, and rank stock footage from platforms like Pexels and Pixabay using deep learning models.

Asynchronous Architecture

This API does not return videos immediately in the POST requests. Because the AI performs heavy operations (LLM query expansion, FAISS semantic matching, and CLIP visual scoring), the API returns a task_id. You must use the GET endpoint to poll for the final results.

Engine Architecture (V16.2)

To integrate perfectly with this API, it is important to understand what happens under the hood when you submit a request.

  • Query Expansion: The engine uses a 70B LLM (via Groq) to break your prompt into 6 cinematic variations.
  • Early Deduplication: Identical videos from Pexels and Pixabay are stripped immediately to save RAM and CPU cycles.
  • FAISS Semantic Mapping: paraphrase-multilingual-MiniLM-L12-v2 compares video tags with your core intent.
  • CLIP Vision Scoring: clip-ViT-L-14 analyzes the actual video frames (thumbnails) to give a visual match percentage.
  • SQLite Zero-RAM Caching: All successful searches are cached on disk for 48 hours. Repeated queries return instantly in 0.1s without hitting the AI.

Authentication

The backend proxy is secured using a static API key. You must include this key in the header of every request.

Security Requirement

Header: x-api-key. If missing or incorrect, the server will return a strict 403 Forbidden response. Do not expose this key in public client-side code without a proxy.

Base URL: The API endpoints should be prefixed with your current host.

Search Single Video POST

/api/search

Submits a visual scene description. The backend spawns a background thread in the ThreadPoolExecutor to process the request safely without blocking the server.

Request Body (JSON)

FieldTypeDescription
query * String The exact visual scene description (e.g., "Cinematic dark forest rain").
orientation String Values: landscape (default), or portrait. Strict aspect ratio filtering is applied.
quality String Values: 1080 (default), 4k (filters out height < 2160), or any.

Example Request

cURL 
curl -X POST /api/search \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_SECRET_KEY" \
  -d '{
    "query": "A lone astronaut walking on Mars",
    "orientation": "landscape",
    "quality": "4k"
  }'

Response (202 Accepted)

You will receive a unique task_id. Save this to poll for status.

{
  "success": true,
  "task_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab"
}

Generate Video Timeline POST

/api/timeline

This powerful endpoint takes a full paragraph/script, splits it into logical sentences (up to 6 scenes), removes stop-words, finds core subjects using NLP, and runs parallel searches for each scene.

Request Body (JSON)

FieldTypeDescription
script * String Your full video script. (e.g., "Space is endless. Stars die and form black holes. Galaxies collide.")
orientation String Values: landscape (default) or portrait.
quality String Values: 1080 (default) or 4k.

Example Request

JavaScript (Fetch)
const response = await fetch('/api/timeline', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': 'YOUR_SECRET_KEY'
  },
  body: JSON.stringify({
    script: "The Roman Empire was vast. Gladiators fought in the colosseum.",
    orientation: "portrait"
  })
});
const data = await response.json();
console.log(data.task_id);

Check Task Status GET

/api/status/{task_id}

Used to fetch the results of your Search or Timeline task. Best Practice: Poll this endpoint every 3 seconds. Tasks remain in memory for 30 minutes before garbage collection.

Path Parameters

task_id * String (UUID) The ID returned by the POST request.

Responses

1. Processing State (200 OK)

{
  "success": true,
  "task": {
    "status": "processing",
    "created_at": 1718293010.12
  }
}

2. Completed State (200 OK) - Search Results

{
  "success": true,
  "task": {
    "status": "completed",
    "count": 12,
    "completed_at": 1718293025.55,
    "data": [
      {
        "id": "pex_456789",
        "source": "Pexels",
        "url": "https://www.pexels.com/video/...",
        "download_url": "https://player.vimeo.com/external/...",
        "thumbnail": "https://images.pexels.com/...",

Data Schemas

The Clip Object

The Engine normalizes responses from both Pixabay and Pexels into a single, unified Clip object for the frontend.

PropertyTypeDescription
idStringPrefixed with pex_ or pix_ to ensure global uniqueness.
download_urlStringDirect MP4 download link of the highest quality available.
thumbnailStringJPG/PNG poster image for the video.
visual_scoreFloat0-100 percentage. Calculated by CLIP-ViT matching frame pixels against the prompt text.
scoreIntegerFinal merit score (Weighted: 50% semantic, 20% resolution, 20% engagement, 10% duration).

Error Codes & Handling

Status CodeError TypeSolution
403 Forbidden Your x-api-key header is missing or incorrect.
400 Bad Request JSON body is missing required parameters (e.g., missing query).
404 Not Found The task_id does not exist or has expired (exceeded 30 minutes TTL).
504 Gateway Timeout The backend HF Space is sleeping. The proxy caught a TimeoutException. Retrying usually wakes it up.
